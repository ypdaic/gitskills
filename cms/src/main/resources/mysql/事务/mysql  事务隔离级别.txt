mysql 的默认隔离级别是可重复读

mysql 隔离级别非4种
1：未提交读（READ UNCOMMITTED）脏读  就是读取了别的事务还没有提交的数据
2：已提交读 （READ COMMITTED）不可重复读  就是读取了别的事务提交了的数据
3：可重复读（REPEATABLE READ） 在我的事务还没关闭的情况下，对某个数据的查询，结果都是一样的，不管其他事务是否对其做过修改
4：可串行化（SERIALIZABLE） 两个事务对同一个数据查询后，不可再次修改或新增


SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED; 设置事物隔离级别


mysql的可重复读通过间隙锁解决了幻读问题（xu）


结论：mysql 的重复读解决了幻读的现象，但是需要 加上 select for update/lock in share mode 变成当前读避免幻读，普通读select存在幻读（是说第一次是

普通select，其他事物更新了数据，如果一次读不是普使用select for update/lock in share mode就会读取别的事物提交的数据，如果第通读，则其他事物不能更新数据）
